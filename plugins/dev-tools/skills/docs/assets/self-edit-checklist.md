# 자가 편집 체크리스트 (Self-Edit Checklist)

초안을 완성한 뒤 출력하기 전에 이 체크리스트를 순서대로 점검한다.

---

## 구조 (Structure)

- [ ] 한 페이지가 한 주제만 다루는가?
- [ ] 첫 번째 헤딩 앞에 개요(overview) 단락이 있는가?
- [ ] 제목에 핵심 키워드가 포함됐는가? (30자 이하)
- [ ] 가치(value)가 기능(feature)보다 앞에 나왔는가?
- [ ] 제목이 질문형이 아닌 서술형인가? ("~란 무엇인가?" → "~의 원리")
- [ ] 헤딩 깊이가 H4(`####`)를 초과하지 않는가?
- [ ] 같은 유형의 다른 문서와 헤딩 패턴이 일관적인가?

---

## 문장 (Sentences)

- [ ] 주어가 명확한가? (누가/무엇이 행동하는가)
- [ ] "~됩니다" 수동태를 능동태로 바꿀 수 있는가?
- [ ] 문장당 하나의 핵심 아이디어만 있는가?
- [ ] 메타 담론을 제거했는가? ("앞서 언급한", "아마 알다시피", "다음 섹션에서는")
- [ ] 동일 개념에 동일 용어를 썼는가? (사용자/유저/클라이언트 혼용 없음)
- [ ] 명사화를 동사로 바꿀 수 있는가? ("설정의 초기화" → "설정을 초기화")
- [ ] 첫 등장 약어를 풀어 썼는가? (JWT → JSON Web Token(JWT))
- [ ] "적절한", "올바른", "관련된" 같은 모호한 수식어를 구체적인 값으로 교체했는가?
- [ ] 번역투를 자연스러운 한국어로 바꿨는가? ("이것은 ~할 수 있게 해줍니다" → "~합니다")

---

## 코드 (Code — 해당되는 경우)

- [ ] 예제 코드를 실제 실행해 검증했는가?
- [ ] 각 코드 블록 아래에 기대 결과(expected output)를 명시했는가?
- [ ] 코드 블록에 언어 식별자가 붙어 있는가? (` ```bash `, ` ```typescript ` 등)
- [ ] 독자가 복사해서 바로 실행할 수 있는 완전한 코드인가? (불완전한 스니펫 없음)

---

## 유형별 추가 점검

**학습 중심 (Tutorial)**:
- [ ] "목표" 섹션이 있는가?
- [ ] 각 단계 완료 후 성공 여부를 확인하는 방법이 있는가?
- [ ] 최종 결과 확인 섹션이 있는가?

**문제 해결 (How-to)**:
- [ ] "결과 확인" 섹션이 있는가?
- [ ] 여러 해결책이 있다면 가장 흔한 원인이 먼저 나왔는가?

**설명 (Explanation)**:
- [ ] 등장 배경 또는 "왜 필요한가" 섹션이 있는가?
- [ ] 추상적인 개념을 시각화(다이어그램, 표)했는가?

**참조 (Reference)**:
- [ ] 오류/예외 케이스를 문서화했는가?
- [ ] Deprecated 항목에 대체 방법을 안내했는가?
