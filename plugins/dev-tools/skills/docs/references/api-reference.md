# 참조 문서 Guide (Reference)

독자가 특정 API·함수·설정의 정확한 스펙을 빠르게 찾아 적용하기 위한 문서.

## 작성 원칙

1. **정확성·완전성 최우선** — 기술적 오류 없음. 항상 최신 상태 유지
2. **일관된 구조** — 개요 → 시그니처 → 매개변수 → 반환값 → 예제 순서 고정
3. **검색·탐색 용이** — 목차, 앵커 링크 활용. 주요 전제 조건(인증, 헤더 등)은 문서 앞에 배치
4. **실행 가능한 예제 필수** — 기본 사용 + 추가 시나리오 포함

## 오류·예외 문서화 패턴

API 레퍼런스는 성공 케이스만큼 오류 케이스를 명확히 다뤄야 한다.

```markdown
## 오류

| 코드 | 상황 | 해결 방법 |
|------|------|----------|
| `400 Bad Request` | `param1`이 빈 문자열일 때 | 유효한 값을 전달하십시오. |
| `401 Unauthorized` | API 키가 없거나 만료됐을 때 | 유효한 API 키를 헤더에 포함하십시오. |
| `429 Too Many Requests` | 요청 한도 초과 | 요청 사이에 1초 이상 간격을 두십시오. |
```

- [ ] 목차 또는 앵커 링크로 탐색이 쉬운가?
